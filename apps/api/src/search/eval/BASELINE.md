## Reading it

- **Floors** (exact-title / exact-content / code / typo) are asserted at 1.0 in
  CI — lexical search has no excuse to miss an exact or fat-fingered term.
- **paraphrase = 0.00 is expected and is the point.** "protect against stolen
  access tokens" shares no lexical tokens with the "Session auth vs JWT" chat;
  only semantic retrieval (phase 3) closes this. The paraphrase row is the honest
  measuring stick for the embedding lift, not a bug.
- Russian/Spanish/mixed score 1.0 here only because the fixture queries reuse
  surface word forms; `simple` FTS has no stemming, so heavily inflected Russian
  will regress on real data — that too is what phase 3 addresses.
